# Typescript 是什么

Typescript（简称 TS） 是 JavaScript（简称 JS）的超集。TS 是基于 JS 构造的一门语言，并不是一门脱离 JS 的全新语言，它在 JS 的基础上增加了一些功能，让你更便捷地编写功能强大的 JS 代码。

那么 TS 语言有个缺点就是，无法直接在 JS 的运行环境中跑起来，例如不能在 Node.js 或者浏览器环境中直接运行 TS 代码，不过 TS 不仅是一门语言，它也是一个编译器，可以把符合 TS 语法的代码编译转化为 JS 代码，然后在 JS 的运行环境里面执行，所以，虽然你写的是 TS 代码，最终在浏览器运行的依然是普通的 JS 代码。

此时就引发一个问题，既然最终运行的是常规 JS 代码，那为何又说 TS 给 JS 增加了额外的功能呢？

答案就是：TS 使用了简洁的语法提供一些高级功能，经过 TS 编译器编译后的 JS 代码片段，有时候非常的复杂难懂，如果你要手写纯 JS 来提供这些功能，那会很痛苦，也难以维护。

另外 TypeScript 还提供一个重要功能，就像它的名字说的那样，提供了『类型定义』，可以很好地帮助开发者在代码执行前就能找到潜在错误。

所以 TS 不仅用简洁语法提供额外功能，同时也提供了错误检测，否则很多错误需要在代码运行时才能发现。

# 为什么使用 Typescript

举个简单例子，下面是一个加法函数

function add (num1, num2) {
    return num1 + num2;
}

console.log( add('1', '2') )

如果给方法传入两个字符串参数，那么结果应该不是你要的，因为两个字符串相加，就是把他们拼接起来，而不是按照数字去做数学运算，最终结果是 12 而不是 3。类似的 JS 错误常常发生在我们代码里面，它不单纯是一个技术错误，并且它运行起来也不会抛出错误，类似这种代码逻辑的错误，未来可能会引发一些可怕的后果。如果用 JS 编写代码，解决的方法通常是在函数的开头去检测过滤非法的参数。
